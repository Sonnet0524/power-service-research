## 核心前提

**驻点工作模式**：流动式服务，一次半天到一天，很久才去第二次

---

## 一、工作特征分析

### 1.1 时间约束

```
【时间约束】半天到一天（约4-8小时）

可完成的工作量估算：
├── 公示、宣传：30分钟
├── 设备检查：1-2小时
├── 客户走访：1-2小时
├── 信息收集：30分钟-1小时
└── 团队协作：可并行推进
```

### 1.2 频率约束

```
【频率约束】每周1-2个小区，很久才去第二次

影响：
├── 不能做需要长期跟踪的工作
├── 不能依赖长期建立的关系
├── 每次都要快速切入
└── 必须在有限时间内抓住重点
```

### 1.3 团队协作

```
【团队协作】几个人一起

优势：
├── 可分工合作
├── 可并行推进
└── 互相配合完成更多工作
```

## 二、工作优先级重新评估

### 2.1 新增评估维度：一次可完成性

| 维度 | 说明 | 评估标准 |
|------|------|---------|
| **一次可完成性** | 能否在一次驻点内完成 | 是/否/部分 |
| **所需时间** | 完成需要多长时间 | 10分钟/30分钟/1小时/2小时 |
| **见效方式** | 效果何时显现 | 即时/短期/长期 |
| **依赖关系** | 是否需要后续跟进 | 不需要/需要/强依赖 |

### 2.2 高优先级工作（重新评估后）

#### 极高优先级（一次见效 + 价值高）

| 工作 | 一次可完成 | 所需时间 | 核心价值 |
|------|-----------|---------|---------|
| **公示客户经理电话** | ✅ 是 | 10分钟 | 服务前置，长期有效 |
| **加入业主群/建立服务群** | ✅ 是 | 20分钟 | 持续服务渠道 |
| **重点客户走访**（频繁停电/敏感客户） | ✅ 是 | 1-2小时 | 主动服务，压降投诉 |
| **配电房检查** | ✅ 是 | 30分钟 | 发现隐患 |
| **发电车接入点查勘** | ✅ 是 | 30分钟 | 应急保供知识库 |

#### 高优先级（一次可完成 + 价值高）

| 工作 | 一次可完成 | 所需时间 | 核心价值 |
|------|-----------|---------|---------|
| **表箱检查** | ✅ 是 | 1小时 | 计量异常发现 |
| **隐患发现上报** | ✅ 是 | 视情况 | 安全管理 |
| **需求收集** | ✅ 是 | 30分钟 | 业务拓展 |
| **物业对接** | ✅ 是 | 30分钟 | 协同基础 |
| **宣传推广** | ✅ 是 | 30分钟 | 服务知晓度 |

#### 中优先级（一次可完成 + 价值中等）

| 工作 | 一次可完成 | 所需时间 | 核心价值 |
|------|-----------|---------|---------|
| **接线图绘制/更新** | ⚠️ 可能需要多次 | 1-2小时 | 知识库建设 |
| **客户信息更新** | ✅ 是 | 30分钟 | 档案完善 |
| **社区对接** | ✅ 是 | 30分钟 | 协同基础 |

#### 低优先级（需要多次或价值较低）

| 工作 | 原因 |
|------|------|
| **长期客户关系维护** | 需要多次，不现实 |
| **深度服务跟踪** | 需要多次，不现实 |
| **复杂设备检修** | 专业性太强 |
| **专业安全检查** | 专业性太强 |

## 三、工作流程设计

### 3.1 驻点前准备（出发前）

```
【驻点前准备】（约15分钟）

├── 1. 查询该小区重点信息
│   ├── 频繁停电客户清单
│   ├── 敏感客户清单
│   ├── 重要客户清单
│   └── 设备信息（如有）
│
├── 2. 确定工作重点
│   ├── 重点走访客户名单
│   ├── 重点检查设备
│   └── 人员分工
│
└── 3. 准备物料
    ├── 公示牌/宣传资料
    ├── 工作表单
    └── 联系方式二维码
```

### 3.2 驻点中执行（半天到一天）

```
【驻点中执行】

├── 【阶段1】到达小区（30分钟）
│   ├── 搭建棚子
│   ├── 张贴公示
│   ├── 加入业主群（如未加入）
│   └── 联系物业配合
│
├── 【阶段2】团队分工推进（2-3小时）
│   │
│   ├── 人员A：宣传接待
│   │   ├── 接待客户咨询
│   │   ├── 收集需求和意见
│   │   └── 宣传推广
│   │
│   ├── 人员B：设备检查
│   │   ├── 配电房检查
│   │   ├── 表箱检查
│   │   ├── 发现隐患
│   │   └── 发电车接入点查勘
│   │
│   └── 人员C：重点客户走访
│       ├── 频繁停电客户走访
│       ├── 敏感客户走访
│       ├── 重要客户走访
│       └── 收集联系方式
│
└── 【阶段3】收尾总结（30分钟）
    ├── 整理收集的信息
    ├── 录入系统（简易记录）
    └── 与物业交接
```

### 3.3 驻点后跟进

```
【驻点后跟进】

├── 1. 信息整理录入（当天完成）
│   ├── 客户信息录入
│   ├── 隐患信息录入
│   ├── 需求信息录入
│   └── 工作记录录入
│
├── 2. 紧急事项处理（当天完成）
│   ├── 严重隐患上报
│   ├── 紧急需求跟进
│   └── 投诉预警上报
│
└── 3. 后续服务（线上）
    ├── 服务群维护
    ├── 问题跟进
    └── 定期信息推送
```

## 四、信息清单的使用方式

### 4.1 驻点前查询

```
【到小区前】通过WPS多维表查询

├── 频繁停电清单
│   ├── 筛选该小区的频繁停电客户
│   ├── 按停电次数排序
│   └── 确定走访优先级
│
├── 敏感客户清单
│   ├── 筛选该小区的敏感客户
│   ├── 了解敏感类型
│   └── 制定服务策略
│
└── 重要客户清单
    ├── 筛选该小区的重要客户
    └── 确定走访安排
```

### 4.2 清单与工作的对应

| 清单类型 | 指导的工作 | 使用时机 |
|---------|-----------|---------|
| 频繁停电清单 | 重点客户走访、设备检查重点 | 驻点前查询 |
| 敏感客户清单 | 服务策略调整、重点走访 | 驻点前查询 |
| 重要客户清单 | 走访安排、服务对接 | 驻点前查询 |
| 设备信息库 | 检查对照、知识积累 | 驻点前/中查询 |

## 五、数字化工具调整

### 5.1 核心需求

```
【流动服务模式的核心需求】

├── 【快速查询】
│   ├── 到小区前快速查询重点信息
│   ├── 移动端友好
│   └── 筛选、排序便捷
│
├── 【信息推送】
│   ├── 去哪个小区就推送该小区重点信息

│   └── 提高效率
│
└── 【一次录入】
    ├── 现场快速记录

```

## 六、核心原则总结

### 6.1 流动服务模式的核心原则

```
┌─────────────────────────────────────────────────────────────┐
│              流动服务模式核心原则                            │
├─────────────────────────────────────────────────────────────┤
│                                                              │
│  1. 【时间有限】必须在半天到一天内完成                        │
│     → 筛选"一次可完成"的工作                                 │
│                                                              │
│  2. 【很久才去第二次】不能依赖长期关系                        │
│     → 信息驱动，而非关系驱动                                 │
│                                                              │
│  3. 【必须抓住重点】时间有限，不能面面俱到                    │
│     → 清单指导，精准聚焦                                     │
│                                                              │
│  4. 【一次见效】每次驻点都要有成效                            │
│     → 公示电话、加入群、重点客户走访                         │
│                                                              │
│  5. 【团队协作】几个人一起，可分工合作                        │
│     → 并行推进，提高效率                                     │
│                                                              │
└─────────────────────────────────────────────────────────────┘
```

### 6.2 与之前认知的对比

| 维度 | 之前认知 | 更新后认知 |
|------|---------|-----------|
| 工作目标 | 全面覆盖 | 精准聚焦 |
| 工作方式 | 长期积累 | 一次见效 |
| 驱动力 | 关系驱动 | 信息驱动 |
| 清单作用 | 系统性指导 | 抓住重点 |
| 数字化重点 | 长期记录 | 快速查询+一次录入 |
