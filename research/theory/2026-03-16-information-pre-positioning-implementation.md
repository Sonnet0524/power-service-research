## 📋 方案概览

**目标**：建立完善的信息前置机制，支撑流动式驻点精准服务

**核心价值**：驻点前推送重点客户清单，实现信息驱动的精准服务

**适用场景**：武侯供电中心现场驻点工作

---

## 一、数据架构设计

### 1.1 数据流架构

```
┌─────────────────────────────────────────────────────┐
│                  数据源层                           │
├─────────────────────────────────────────────────────┤
│ 营销系统 │ PMS系统 │ 95598系统 │ 驻点记录系统      │
└─────────────────────────────────────────────────────┘
                         ↓
┌─────────────────────────────────────────────────────┐
│                  数据汇聚层                         │
├─────────────────────────────────────────────────────┤
│        WPS多维表：小区重点客户统计表                │
└─────────────────────────────────────────────────────┘
                         ↓
┌─────────────────────────────────────────────────────┐
│                  数据应用层                         │
├─────────────────────────────────────────────────────┤
│ 驻点前信息推送 │ 现场快速查询 │ 驻点后记录录入     │
└─────────────────────────────────────────────────────┘
```

### 1.2 核心数据表

| 3 | 小区地址 | 文本 | 是 | 营销系统 | 详细地址 |
| 4 | 总户数 | 数字 | 是 | 营销系统 | 小区总户数 |
| 5 | 商户数 | 数字 | 是 | 营销系统 | 小区商户数 |
| 6 | 配电房位置 | 文本 | 是 | PMS系统 | 如"地下停车场B1层" |
| 7 | 配电房数量 | 数字 | 是 | PMS系统 | 配电房数量 |
| 8 | 客户经理 | 文本 | 是 | 组织架构 | 负责客户经理姓名 |

| 11 | 社区联系人 | 文本 | 否 | 手工录入 | 社区联络人姓名 |
| 12 | 社区联系电话 | 文本 | 否 | 手工录入 | 社区联络人电话 |
| 13 | 是否建立微信群 | 选择 | 是 | 驻点记录 | 是/否 |

| 17 | 备注 | 文本 | 否 | 手工录入 | 特殊说明 |

**表名**：`客户台账`

**用途**：记录重点客户信息，支撑精准服务

**字段设计**：

| 序号 | 字段名 | 字段类型 | 是否必填 | 数据来源 | 说明 |

| 3 | 联系电话 | 文本 | 是 | 营销系统 | 客户电话 |
| 4 | 用电地址 | 文本 | 是 | 营销系统 | 详细地址 |
| 5 | 小区名称 | 关联 | 是 | 小区台账 | 关联小区 |
| 6 | 客户类型 | 选择 | 是 | 系统标记 | 重要客户/敏感客户/频繁停电客户/特殊群体/普通客户 |
| 7 | 重要性等级 | 选择 | 是 | 系统评估 | 高/中/低 |

| 12 | 上次投诉时间 | 日期 | 否 | 95598系统 | 最近一次投诉时间 |

| 14 | 走访次数 | 数字 | 否 | 驻点记录 | 累计走访次数 |

| 16 | 备注 | 文本 | 否 | 手工录入 | 特殊说明 |

**客户类型说明**：

| 客户类型 | 定义 | 识别规则 | 服务重点 |
|---------|------|---------|---------|
| **重要客户** | 重要电力用户 | 系统标记 | 优先服务、重点保障 |
| **敏感客户** | 近期有投诉或诉求 | 近12月投诉次数≥1 | 关注感受、主动沟通 |
| **频繁停电客户** | 经常停电的客户 | 近12月停电次数≥3 | 解释说明、优先抢修 |
| **特殊群体** | 老人、残疾人等 | 手工标记 | 上门服务、亲情关怀 |
| **普通客户** | 其他客户 | 默认值 | 标准服务 |

## 二、信息推送机制设计

### 2.1 推送触发规则

**触发条件**：
- 驻点前1天（可配置）
- 新增驻点计划时

**推送对象**：
- 负责该小区的客户经理

**推送渠道**：
- 企业微信（主要渠道）

## 三、快速查询功能设计

### 3.1 查询场景设计

**场景1：客户信息查询**

**触发**：客户经理在驻点现场需要了解客户情况

**操作流程**：
```
1. 打开企业微信小程序/WPS多维表移动端
2. 选择"客户信息查询"
3. 输入客户编号/电话/地址（支持模糊搜索）
4. 查看客户详细信息
   - 基本信息：姓名、电话、地址
   - 客户类型：重要客户/敏感客户/频繁停电客户/特殊群体
   - 服务记录：上次走访时间、服务内容
   - 停电历史：近12月停电次数、上次停电时间
   - 投诉历史：近12月投诉次数、上次投诉时间
```

**场景2：配电房信息查询**

**触发**：客户经理需要检查配电房

**操作流程**：
```
1. 打开企业微信小程序/WPS多维表移动端
2. 选择"配电房信息查询"
3. 选择小区名称
4. 查看配电房详细信息
   - 配电房位置
   - 配电房数量
   - 设备信息（变压器信息、供电方式）
   - 上次检查时间
   - 隐患记录
   - 现场照片
```

**场景3：应急资源查询**

**触发**：客户经理需要应急处置

**操作流程**：
```
1. 打开企业微信小程序/WPS多维表移动端
2. 选择"应急资源查询"
3. 选择小区名称
4. 查看应急资源信息
   - 应急发电车接入点
   - 抢修设备位置
   - 应急联系人
   - 应急物资储备
   - 应急通道
```

## 四、一次录入机制设计

### 4.1 录入场景设计

**场景**：客户经理驻点结束，需要记录工作情况

**录入原则**：
- **一次录入**：所有信息只录入一次，不重复录入
- **自动填充**：自动填充小区名称、客户经理等信息
- **必填最少**：只保留必要的必填字段
- **快速完成**：整个录入过程不超过5分钟

### 4.3 自动化联动机制


**联动2：驻点记录 → 小区台账**

## 五、数据维护机制

### 5.1 数据更新规则

| 数据类型 | 更新频率 | 更新方式 | 负责人 |
|---------|---------|---------|--------|
| **小区基础信息** | 新增小区时 | 手工录入 | 客户经理 |
| **重点客户信息** | 实时 | 系统自动+手工维护 | 系统自动 |
| **驻点工作记录** | 每次驻点后 | 手工录入 | 客户经理 |
| **停电次数** | 每周 | 从PMS系统同步 | 系统自动 |
| **投诉次数** | 实时 | 从95598系统同步 | 系统自动 |

### 5.2 数据质量保障

**机制1：必填字段校验**
- 驻点工作记录的关键字段设为必填
- 未填写完整无法保存

**机制2：数据格式校验**
- 电话号码格式校验
- 日期格式校验
- 数字范围校验

**机制3：数据关联校验**
- 走访客户清单必须是已存在的客户编号
- 小区名称必须是已存在的小区

**机制4：数据完整性检查**
- 定期检查重点客户信息完整性
- 定期检查小区基础信息完整性

## 八、风险与应对

### 8.1 主要风险

| 风险 | 影响 | 概率 | 应对措施 |
|------|------|------|---------|
| **数据不准确** | 推送信息错误 | 中 | 建立数据校验机制，定期核查数据 |
| **推送不及时** | 信息前置失效 | 低 | 建立自动化推送机制，设置提醒 |
| **客户经理不使用** | 机制形同虚设 | 中 | 加强培训，简化操作，纳入考核 |
| **系统不稳定** | 无法正常使用 | 低 | 选择稳定的平台，建立备份机制 |
| **数据安全风险** | 客户信息泄露 | 中 | 设置权限控制，加强数据安全管理 |

### 8.2 应对措施

**措施1：数据质量保障**
- 建立数据校验机制
- 定期核查数据准确性
- 设置数据质量责任人

**措施2：推送及时性保障**
- 建立自动化推送机制
- 设置推送提醒
- 建立推送失败告警

**措施3：使用率提升**
- 加强培训，确保客户经理会用
- 简化操作，确保客户经理愿意用
- 纳入考核，确保客户经理必须用

**措施4：系统稳定性保障**
- 选择稳定的平台（WPS、企业微信）
- 建立数据备份机制
- 制定应急预案

**措施5：数据安全保障**
- 设置权限控制，只有授权人员可访问
- 加强数据安全管理，定期检查权限
- 建立数据安全事件应急预案
