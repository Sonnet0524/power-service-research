## 核心问题

**信息和数据的价值是什么？如何安全地使用？**

---

## 一、信息流向模型

### 1.1 信息分类

```
信息流向模型
│
├── 【输入信息】公司 → 客户经理
│   ├── 频繁停电清单（供电质量预警）
│   ├── 敏感客户清单（服务质量预警）
│   ├── 重要客户清单（保供电重点）
│   ├── 欠费预警清单
│   └── 业扩预警清单
│
└── 【输出信息】客户经理 → 公司
    ├── 设备信息（位置、照片、状态）
    ├── 隐患信息（位置、描述、照片）
    ├── 客户联系方式（现场获取）
    ├── 需求信息（业扩、新业务）
    └── 诉求信息（意见、建议）
```

### 1.2 信息价值对比

| 类型 | 方向 | 客户经理角色 | 核心价值 |
|------|------|-------------|---------|
| 输入信息 | 公司→客户经理 | 消费方 | 指导驻点工作 |
| 输出信息 | 客户经理→公司 | 收集者 | 支撑其他工作 |

## 四、数据安全方案

### 4.1 核心约束

```
【约束一】内外网隔离
├── 内网与互联网物理/逻辑隔离
├── 无数据穿透通道
└── 数据交换需人工传递

【约束二】敏感信息保护
├── 客户信息属于敏感信息
├── 不能大规模完整在互联网存放
└── 需要相应机制避免泄露

【约束三】工作需要
├── 客户经理现场需要完整信息
├── 无具体地址和电话无法服务
└── 清单必须具有实际使用价值
```

### 4.2 安全与效率的平衡

```
┌─────────────────────────────────────────────────────────────┐
│                安全与效率的平衡点                            │
├─────────────────────────────────────────────────────────────┤
│                                                              │
│  【提供内容】完整信息（工作需要）                             │
│  ├── 客户编号                                               │
│  ├── 客户姓名                                               │
│  ├── 联系电话                                               │
│  ├── 用电地址（具体到门牌）                                  │
│  └── 相关标签信息                                           │
│                                                              │
│  【安全措施】最小范围提供                                    │
│  ├── 区域隔离：客户经理只看本区域                            │
│  ├── 人员限制：只给需要的人                                  │
│  ├── 字段权限：敏感字段严格控制                              │
│  ├── 禁止导出：防止批量获取                                  │
│  └── 访问审计：所有行为可追溯                                │
│                                                              │
│  【核心原则】                                                │
│  信息完整提供，但范围最小化                                  │
│  让客户经理能工作，但严格控范围                              │
│                                                              │
└─────────────────────────────────────────────────────────────┘
```

### 4.3 数据更新机制

```
【输入信息：内网 → 互联网】

内网系统导出 ──→ 按区域拆分 ──→ 人工上传 ──→ WPS多维表
                    │
                    └── 武侯区、锦江区等分别上传

【更新频率】
├── 频繁停电清单：每日更新
├── 敏感客户清单：每日更新
└── 重要客户清单：每周更新

【输出信息：互联网 → 内网】

WPS多维表 ──→ 定期导出 ──→ 人工导入 ──→ 内网系统

【同步频率】
├── 设备信息：每周同步
├── 客户联系方式：每周同步
├── 隐患信息：每日同步
└── 需求信息：每周同步
```

## 五、权限控制方案

### 5.1 WPS多维表权限设置

```
【权限分级】

🔴 完全权限（区域管理员）
├── 查看、编辑、导出
├── 管理本区域数据
└── 适用：区域负责人

🟠 编辑权限（客户经理）
├── 查看本区域完整信息
├── 编辑自己录入的数据
├── 不能导出完整清单
└── 适用：客户经理

🟡 只读权限（其他人员）
├── 查看脱敏信息
├── 不能查看敏感字段
└── 适用：辅助人员

【区域隔离】
├── 通过WPS多维表筛选权限实现
├── 客户经理A：区域="武侯区某片区"
├── 客户经理B：区域="锦江区某片区"
└── 每人只能看到自己负责的行
```

### 5.2 安全保障措施

| 措施 | 实施方式 | 目的 |
|------|---------|------|
| **区域隔离** | 按区域分配权限 | 客户经理只看本区域 |
| **字段权限** | 敏感字段仅授权可见 | 最小权限原则 |
| **访问审计** | WPS访问日志 | 追溯访问行为 |
| **禁止导出** | 限制导出权限 | 防止批量泄露 |
| **截图水印** | 显示访问者信息 | 威慑泄露行为 |
| **定期清理** | 过期数据删除 | 减少风险暴露 |
| **保密协议** | 签署协议 | 法律约束 |
| **安全培训** | 定期培训 | 提高意识 |

## 七、关键结论

### 7.1 信息价值

1. **输入信息**指导驻点工作，**输出信息**支撑其他工作
2. 频繁停电清单和敏感客户清单具有**极高预警价值**
3. 两类清单交叉分析可识别**极高风险客户**

### 7.2 数据安全

1. **完整信息+最小范围**是平衡安全与效率的关键
2. 通过**区域隔离、权限控制、访问审计**保障安全
3. 内外网隔离环境下，采用**人工传递**方式更新数据

### 7.3 数字化赋能

1. 以**企业微信+WPS**为核心工具
2. 不开发复杂APP，利用现有平台
3. 智能体辅助咨询和诊断
