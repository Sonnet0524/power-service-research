| **配电房位置** | PMS系统/现场 | 首次驻点 | 配电房检查 | 客户经理 | 快速定位 |
| **配电房数量** | PMS系统/现场 | 首次驻点 | 工作量评估 | 客户经理 | 规划时间 |

| **物业电话** | 现场采集 | 首次驻点 | 日常联系 | 客户经理 | 协调工作 |

**关键使用场景示例**：

**场景1：驻点前规划**
```
使用信息：
- 小区名称、地址 → 导航到达
- 总户数、商户数 → 评估工作量和重点
- 配电房位置、数量 → 规划用电检查时间
- 上次驻点时间 → 确定本次重点
- 是否建立微信群 → 确认是否需要入群

使用时机：驻点前1天
使用者：客户经理
使用目的：制定驻点工作计划
```

**场景2：社网共建对接**
```
使用信息：
- 社区名称、联系人 → 对接对象
- 小区地址 → 确定网格范围
- 驻点次数 → 展示服务记录

使用时机：社网共建对接时
使用者：客户经理/供电所
使用目的：建立合作关系
```

**关键使用场景示例**：

**场景1：驻点前获取重点客户清单**
```
系统自动推送信息：
- 敏感客户（近12月投诉≥1次）
  使用信息：客户姓名、电话、地址、上次投诉时间、投诉内容
  使用时机：驻点前1天
  使用者：客户经理
  使用目的：主动回访，了解满意度

- 频繁停电客户（近12月停电≥3次）
  使用信息：客户姓名、电话、地址、停电次数、上次停电时间
  使用时机：驻点前1天
  使用者：客户经理
  使用目的：解释说明，优先保障

- 特殊群体（独居老人、残疾人等）
  使用信息：客户姓名、电话、地址、特殊类型、备注
  使用时机：驻点前1天
  使用者：客户经理
  使用目的：上门关怀，安全检查
```

**场景2：现场快速查询客户信息**
```
触发：客户经理在驻点现场需要了解某客户情况

使用信息：
- 基本信息：姓名、电话、地址
- 客户类型：重要客户/敏感客户/频繁停电客户/特殊群体
- 服务历史：上次走访时间、走访次数、服务记录
- 用电历史：停电次数、投诉次数

使用时机：驻点现场
使用者：客户经理
使用目的：快速了解客户，针对性服务
```

**场景3：制定差异化服务策略**
```
使用信息：客户类型、重要性等级、敏感类型

策略制定：
- 敏感客户 → 主动回访、优先响应、重点关注
- 频繁停电客户 → 解释说明、优先抢修、定期沟通
- 特殊群体 → 上门服务、亲情关怀、安全检查
- 重要客户 → 定期走访、优先保障、专属服务

使用时机：服务规划时
使用者：客户经理
使用目的：提供差异化、精准化服务
```

## 五、信息安全管理

### 5.1 信息分级

| 信息级别 | 信息内容 | 访问权限 | 使用限制 |
|---------|---------|---------|---------|
| **公开级** | 小区名称、地址、公示信息 | 所有人 | 无限制 |
| **内部级** | 客户姓名、设备信息、工作记录 | 内部员工 | 工作需要 |
| **敏感级** | 客户电话、投诉记录、敏感客户信息 | 授权人员 | 审批使用 |
| **机密级** | 系统配置、密钥 | 管理员 | 严格管控 |

### 5.2 使用规范

1. **最小化原则**：只采集必要的信息，不过度采集
2. **授权使用**：敏感信息需经授权方可使用
3. **工作目的**：信息只能用于工作目的，不得他用
4. **安全存储**：信息存储需加密，定期备份
5. **及时更新**：过时信息及时更新或删除
