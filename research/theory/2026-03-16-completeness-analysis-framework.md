## 理论框架：核心-扩展双层模型

### 模型定义

```
┌─────────────────────────────────────────────────────────┐
│                    驻点工作体系                          │
├─────────────────────────────────────────────────────────┤

│  │            核心层（必须有）                       │   │
│  │                                                  │   │
│  │  特征：直接支撑三大前置目标                       │   │
│  │  考核：必须量化考核                               │   │

│  └─────────────────────────────────────────────────┘   │

│  └─────────────────────────────────────────────────┘   │
└─────────────────────────────────────────────────────────┘
```

### 模型核心思想

**驻点工作的本质是"前置"**：

| 前置类型 | 核心领域 | 核心目标 |
|---------|---------|---------|
| 服务前置 | 网格服务 | 让客户找到人，压降95598工单 |
| 信息前置 | 用电检查 | 掌握设备状况，消除安全隐患 |
| 资源前置 | 应急保供 | 建立知识库，支撑快速处置 |

## 分析方法：先扩展再收缩

### 第一步：扩展思考

**方法**：不加限制地思考客户经理到驻点现场所有可能做的工作

**维度**：
1. 客户接触类
2. 设备设施类
3. 应急保供类
4. 信息数据类
5. 关系协调类
6. 安全管控类
7. 业务拓展类

**结果**：梳理出约145项可能的工作内容

### 第二步：责任边界标注

**分类框架**：

| 专业领域 | 说明 |
|---------|------|
| 营销服务类 | 客户经理核心专业 |
| 用电检查类 | 客户经理相关专业 |
| 配电运维类 | 非客户经理专业 |
| 应急保供类 | 跨专业协同 |
| 安全管理类 | 非客户经理专业 |
| 市场拓展类 | 营销相关 |
| 综合协调类 | 客户经理相关工作 |

**职责属性**：

| 属性 | 定义 | 特征 |
|------|------|------|
| ⭐ 主要职责 | 客户经理本专业、本职工作 | 直接对结果负责 |
| 🔹 辅助职责 | 非本专业，但可提供辅助支持 | 发现、收集、提醒 |
| 🔸 协助职责 | 配合其他专业开展工作 | 配合、协调、信息支撑 |
| ⚪ 非职责 | 其他专业岗位的职责 | 专业执行、管理决策 |

### 第三步：可行性与价值分析

**信息收集类评估维度**：
- 收集难度：低/中/高
- 信息价值：低/中/高
- 数字化支撑：有/无/可开发
- 综合推荐：高/中/低

**操作类评估维度**：
- 操作难度：低/中/高
- 资质要求：无/简单培训/专业资质
- 执行频率：高频/中频/低频
- 数字化支撑：有/无/可开发
- 综合推荐：高/中/低

## 完备性分析结论

### 对核心问题的回答

**Q1：驻点工作是否只有这3个领域？**

> **答案**：当前清单覆盖的是**核心层**工作，从"三大前置"目标看是完备的。但存在**扩展层**工作可以补充。

**Q2：客户经理职责边界是什么？**

> **答案**：
> - **主要职责**（⭐）：营销服务、应急保供知识库建设、综合协调
> - **辅助职责**（🔹）：用电检查相关专业
> - **协助职责**（🔸）：配电运维、安全管理等
> - **非职责**（⚪）：专业执行、管理决策类

**Q3：驻点工作是"全面覆盖"还是"重点聚焦"？**

> **答案**：**重点聚焦**。驻点工作的本质是"前置"，核心是做好三大前置，扩展层是"顺便做"。

### 完备性评估

| 评估维度 | 结论 | 说明 |
|---------|------|------|
| 核心工作完备性 | ✅ 完备 | 三大领域覆盖核心前置目标 |
| 扩展工作补充 | ⚠️ 可补充 | 信息收集、问题排查等可列入扩展层 |
| 工作边界清晰度 | ✅ 已明确 | 已标注职责属性 |
