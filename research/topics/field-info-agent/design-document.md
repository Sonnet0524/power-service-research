# 现场信息收集智能体设计

## 一、需求分析：理解现场工作场景

### 1.1 现场工作痛点

基于Session-005的实践反馈，现场人员面临的主要问题：

**信息收集痛点**：
- ❌ 手工填写表单效率低
- ❌ 纸质记录容易遗漏
- ❌ 现场网络不稳定影响数据上传
- ❌ 信息采集标准不统一
- ❌ 照片与文字描述分离，后期整理困难

**文档生成痛点**：
- ❌ 回到办公室后需要重新整理
- ❌ 多源信息（纸质笔记、照片、记忆）难以整合
- ❌ 报告格式不统一，质量参差不齐
- ❌ 耗时较长，影响工作效率

**知识沉淀痛点**：
- ❌ 信息分散在个人手中
- ❌ 难以形成可复用的知识库
- ❌ 经验传承困难

### 1.2 智能体能解决什么

**核心目标**：让现场人员"开口说、随手拍"，剩下的事交给Agent

| 人工环节 | Agent辅助 | 价值 |
|---------|----------|------|
| 填写纸质表单 | 语音输入自动转结构化数据 | 效率提升3-5倍 |
| 拍照+手写备注 | 拍照自动关联+语音说明 | 信息完整性提升 |
| 现场记忆整理 | 实时记录，即时确认 | 准确率提升 |
| 回办公室写报告 | 现场自动生成初稿 | 即时交付 |
| 信息录入系统 | 一键同步到知识库 | 知识沉淀 |

### 1.3 信息采集场景拆解

基于"三个前置"框架，现场需要采集的信息：

#### 场景A：配电房信息采集
**采集内容**：
- 配电房位置（GPS+文字描述）
- 变压器信息（型号、容量、投运时间）
- 设备台账（开关柜、电缆）
- 技术图纸（拍照存档）
- 隐患记录（描述+照片）

**智能体辅助**：
- 语音描述自动识别关键信息
- 拍照自动分类存档
- 结构化数据即时生成

#### 场景B：发电车应急接入信息采集
**采集内容**：
- 进入点（车辆入口位置）
- 停放点（发电车停放位置）
- 接入点（接入配电系统的具体位置）
- 电缆信息（型号、长度）
- 现场照片（全景+细节）

**智能体辅助**：
- 引导式采集（按清单逐项确认）
- 拍照自动标注位置信息
- 生成应急接入方案初稿

#### 场景C：客户走访信息采集
**采集内容**：
- 客户基本信息（自动关联台账）
- 走访时间、方式
- 了解情况（用电是否正常、是否有问题）
- 客户反馈和诉求
- 处理措施
- 客户满意度

**智能体辅助**：
- 自动调取客户历史信息
- 语音转文字记录对话要点
- 自动生成走访记录表

#### 场景D：表箱设备缺陷采集
**采集内容**：
- 表箱位置（地址+表号）
- 缺陷类型（设备老化、线路破损等）
- 缺陷等级（紧急/重要/一般）
- 缺陷照片
- 整改建议

**智能体辅助**：
- 拍照识别表号（OCR）
- 缺陷类型智能分类
- 自动生成缺陷报告

## 二、智能体架构设计

### 2.1 核心能力模型

```
┌─────────────────────────────────────────────────────┐
│                 Field Info Agent                     │
│               现场信息收集智能体                      │
├─────────────────────────────────────────────────────┤
│  感知层  │  理解层   │  生成层   │  输出层          │
├─────────┼──────────┼──────────┼─────────────────┤
│ 语音输入 │ 语义理解 │ 数据结构化│ 表单生成        │
│ 拍照输入 │ 信息抽取 │ 内容生成  │ 报告生成        │
│ 手写输入 │ 逻辑校验 │ 知识关联  │ 知识库更新      │
│ GPS定位 │ 场景识别 │ 自动分类  │ 系统同步        │
└─────────┴──────────┴──────────┴─────────────────┘
```

### 2.2 能力详解

#### 感知层：多模态输入
- **语音输入**：实时语音转文字，支持方言识别
- **拍照输入**：高清照片拍摄，自动压缩和标注
- **手写输入**：关键信息手写补充
- **GPS定位**：自动记录位置信息
- **扫码输入**：扫描设备二维码获取基础信息

#### 理解层：智能解析
- **语义理解**：理解语音描述的意图和内容
- **信息抽取**：从描述中提取结构化信息
- **逻辑校验**：检查信息的完整性和一致性
- **场景识别**：自动识别当前采集场景类型

#### 生成层：内容处理
- **数据结构化**：将非结构化输入转为标准格式
- **内容生成**：基于模板生成文档初稿
- **知识关联**：关联已有知识库信息
- **自动分类**：按标准分类体系组织信息

#### 输出层：多格式交付
- **表单生成**：标准信息采集表单
- **报告生成**：工作总结、巡检报告
- **知识库更新**：自动更新小区台账、设备台账
- **系统同步**：与企业微信、供电所系统对接

## 三、人机协作流程设计

### 3.1 工作模式：副驾驶模式

智能体作为"副驾驶"，现场人员始终掌控：

```
现场人员（主导）          Agent（辅助）
    │                         │
    │  启动采集任务           │
    ├────────────────────────>│
    │                         │
    │  语音/拍照输入          │
    ├────────────────────────>│
    │                         │  实时处理
    │  查看Agent理解和建议    │<────────────────────────┤
    │<────────────────────────│
    │                         │
    │  确认或修正             │
    ├────────────────────────>│
    │                         │
    │  完成采集               │
    ├────────────────────────>│
    │                         │  生成文档
    │  预览和确认文档         │<────────────────────────┤
    │<────────────────────────│
    │                         │
    │  提交/保存              │
    └────────────────────────>│
                              │  更新知识库
```

### 3.2 典型工作流程

#### 流程A：配电房信息采集

**Step 1: 任务启动**
- 现场人员："开始配电房信息采集"
- Agent：调取小区基本信息，加载采集模板

**Step 2: 信息引导**
- Agent："请描述配电房位置"
- 现场人员：语音描述位置
- Agent：转文字，提取地址信息，确认GPS坐标

**Step 3: 设备信息**
- Agent："请拍摄变压器铭牌"
- 现场人员：拍照
- Agent：OCR识别铭牌信息，自动填充型号、容量

**Step 4: 隐患记录**
- Agent："是否有设备隐患？"
- 现场人员：语音描述隐患
- Agent：结构化记录（类型、位置、描述）
- 现场人员：拍摄隐患照片
- Agent：照片自动关联到隐患记录

**Step 5: 完成确认**
- Agent：展示采集的信息清单
- 现场人员：确认或补充
- Agent：生成分配电房信息表

**Step 6: 知识库更新**
- 一键同步到小区台账
- 更新设备信息库
- 创建隐患整改台账

#### 流程B：客户走访

**Step 1: 客户识别**
- 现场人员：扫描客户二维码或输入客户编号
- Agent：自动调取客户历史信息
  - 上次走访时间
  - 历史投诉记录
  - 用电基本信息

**Step 2: 走访记录**
- Agent：引导式提问
  - "近期用电是否正常？"
  - "是否有用电问题需要解决？"
  - "对服务是否满意？"
- 现场人员：语音回答
- Agent：实时转录，提取关键信息

**Step 3: 诉求处理**
- 如有诉求，Agent：
  - 分类诉求类型
  - 建议处理措施
  - 记录处理结果

**Step 4: 生成记录**
- 自动生成《客户走访记录表》
- 包含：客户信息、走访内容、处理措施、满意度

### 4.2 关键技术选型

| 技术领域 | 推荐方案 | 说明 |
|---------|---------|------|
| **语音识别** | 百度语音识别 / 科大讯飞 | 支持中文、方言识别 |
| **图像识别** | PaddleOCR | 开源、中文优化好 |
| **自然语言处理** | LLM（大语言模型） | 语义理解、信息抽取 |
| **知识图谱** | Neo4j / 关系型数据库 | 小区-设备-客户关联 |
| **移动端** | 微信小程序 / 企业微信应用 | 便于推广使用 |
| **后端** | Python + FastAPI | 轻量、高效 |
| **数据库** | PostgreSQL + 对象存储 | 结构化+非结构化 |

## 五、核心功能设计

### 5.1 智能语音助手

**功能描述**：
- 实时语音转文字
- 语义理解，自动填充表单
- 多轮对话引导

**示例对话**：
```
用户："配电房在小区3号楼地下室"
Agent："收到，配电房位置：3号楼地下室。GPS坐标已记录。请拍摄配电房入口照片。"

用户：[拍照]
Agent："照片已保存。现在请描述变压器情况。"

用户："有两台变压器，一台是2015年投运的，容量500千伏安"
Agent："识别到信息：
- 变压器数量：2台
- 投运时间：2015年
- 容量：500kVA
是否需要补充型号信息？"
```

### 5.2 智能拍照助手

**功能描述**：
- 拍照自动分类
- OCR识别关键信息
- 照片与数据自动关联

**示例**：
- 拍摄变压器铭牌 → OCR识别型号、容量、厂家
- 拍摄表箱 → OCR识别表号
- 拍摄隐患 → 自动关联隐患描述

### 5.3 智能表单生成

**功能描述**：
- 基于采集数据自动生成标准表单
- 支持多种格式（Word、PDF、Excel）
- 一键导出

**生成表单类型**：
1. 配电房信息表
2. 发电车应急接入信息表
3. 客户走访记录表
4. 隐患整改台账
5. 工作总结报告

### 5.4 知识库自动构建

**功能描述**：
- 采集信息自动入库
- 自动建立关联关系
- 支持全文检索

**示例**：
- 采集某小区配电房信息 → 自动更新小区台账
- 记录客户走访 → 更新客户档案
- 发现隐患 → 创建隐患台账，关联设备信息

## 六、实施路径

### 阶段1：MVP开发（2-3周）
- 核心功能：语音输入 + 基础表单生成
- 覆盖场景：配电房信息采集
- 技术验证：语音识别准确率、表单生成质量

### 阶段2：功能扩展（2-3周）
- 增加拍照OCR功能
- 增加客户走访场景
- 完善知识库功能

### 阶段3：试点验证（2-4周）
- 选择1-2个供电所试点
- 收集用户反馈
- 迭代优化

### 阶段4：全面推广
- 完善所有场景
- 培训推广
- 持续优化
