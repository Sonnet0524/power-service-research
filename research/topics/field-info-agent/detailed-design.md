# 现场信息收集Agent - 详细设计方案

## 一、基本约束（必须遵守）

### 1.1 数据权限规则

**原则**: 供电所内成果共享

```yaml
权限模型:
  数据可见性:
    - 本供电所人员: 可查看、可编辑本供电所所有数据
    - 其他供电所人员: 不可见
    - 供电所长/管理员: 可查看全所数据，可导出报表
  
  操作权限:
    - 采集人员: 创建、编辑自己采集的记录
    - 供电所全员: 查看所有记录，可评论、补充
    - 所长: 审核、归档、删除权限

实施方式:
  - WPS多维表格按供电所分表或在表中增加"供电所"字段
  - 文档存储路径包含供电所标识
  - 企业微信应用按供电所隔离（每个供电所独立应用）
```

### 1.3 网络要求

**原则**: 必须在线使用（企业微信依赖网络）

```yaml
网络依赖说明:
  - 企业微信消息接收: 必须联网
  - WPS云文档操作: 必须联网
  - 语音识别: 可本地或云端，建议云端（准确率高）
  - 图片上传: 必须联网
  
离线场景提示:
  - 如网络中断，Agent提示"网络异常，请检查网络后重试"
  - 不支持离线缓存（简化设计）
  - 现场人员需确保手机有信号或使用移动数据
```

## 二、通知规则设计

### 2.1 通知类型与触发条件

```yaml
通知体系:

  # 类型1: 工作完成通知
  work_completed:
    trigger: "驻点工作完成后"
    receivers: 
      - 采集人员本人（确认成功）
      - 供电所长（知晓完成）
    content: "{小区名}驻点工作已完成，采集{N}项数据，生成{M}份文档"
    priority: normal
    
  # 类型2: 问题发现通知
  issue_found:
    trigger: "发现设备缺陷或安全隐患时"
    receivers:
      - 采集人员本人
      - 供电所长（立即）
      - 设备管理专责（缺陷类）
    content: "发现{问题类型}：{问题描述}，位置：{具体位置}，建议立即处理"
    priority: high
    
  # 类型3: 日报汇总
  daily_report:
    trigger: "每日18:00"
    receivers:
      - 供电所长
      - 当日有驻点工作的人员
    content: "今日驻点工作总结：共{N}人开展驻点工作，覆盖{M}个小区，发现{X}个问题"
    priority: low
    schedule: "0 18 * * *"  # 每天18点
    
  # 类型4: 周报汇总
  weekly_report:
    trigger: "每周五18:00"
    receivers:
      - 供电所长
      - 全体客户经理
    content: "本周驻点工作周报：..."
    priority: normal
    schedule: "0 18 * * 5"  # 每周五18点
    
  # 类型5: 应急事件通知
  emergency_event:
    trigger: "启动应急处置流程时"
    receivers:
      - 现场人员
      - 供电所长（立即）
      - 抢修队伍负责人
      - 分管领导（重大事件）
    content: "应急事件：{事件类型}，地点：{小区名}，请立即响应"
    priority: urgent
    
  # 类型6: 文档生成完成
  doc_generated:
    trigger: "文档自动生成完成"
    receivers:
      - 采集人员本人
    content: "{文档类型}已生成：{文档链接}"
    priority: normal
    
  # 类型7: 待办提醒
  todo_reminder:
    trigger: "有未完成的驻点工作时"
    condition: "距离计划驻点时间超过24小时未开始"
    receivers:
      - 计划执行人
    content: "提醒：您计划的{小区名}驻点工作尚未开始，计划时间：{日期}"
    priority: normal
```

### 2.2 通知实现方式

```yaml
企业微信通知渠道:

  # 方式1: 应用消息（主要方式）
  app_message:
    target: 单聊（发给个人）
    type: text / markdown / 图文卡片
    used_for:
      - 工作完成通知
      - 文档生成完成
      - 待办提醒
      
  # 方式2: 群机器人（次要方式）
  group_bot:
    target: 供电所工作群
    type: markdown
    used_for:
      - 日报汇总
      - 周报汇总
      - 问题发现通知
      
  # 方式3: 企业微信卡片消息（重要通知）
  card_message:
    target: 单聊
    type: 模板卡片
    used_for:
      - 应急事件通知
      - 重大问题发现
    features:
      - 可点击跳转
      - 可显示关键信息
      - 支持操作按钮（如"查看详情"、"确认收到"）
```

### 4.2 权限过滤实现

```javascript
// 所有查询自动添加供电所过滤
async function queryWithPermission(tableName, filter, userInfo) {
  // 获取用户所属供电所
  const powerStation = await getUserPowerStation(userInfo.userId);
  
  // 添加供电所过滤条件
  const permissionFilter = {
    ...filter,
    供电所ID: {
      operator: 'equals',
      value: powerStation.id
    }
  };
  
  // 执行查询
  return await wps.query(tableName, permissionFilter);
}

// 文档访问权限
async function getDocumentPermission(docUrl, userInfo) {
  // 从URL解析供电所
  const docPowerStation = parsePowerStationFromUrl(docUrl);
  
  // 获取用户供电所
  const userPowerStation = await getUserPowerStation(userInfo.userId);
  
  // 权限检查
  if (docPowerStation !== userPowerStation.id) {
    return { allowed: false, reason: '无权限访问其他供电所文档' };
  }
  
  return { allowed: true };
}
```
