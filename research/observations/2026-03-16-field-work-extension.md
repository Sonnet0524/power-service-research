## 🤔 思考框架

### 从"客户经理在现场能做什么"角度出发

客户经理到达居民小区现场后，除了当前清单中的工作，还可能开展哪些工作？

## 🔍 可能缺失的工作内容

### 一、信息收集类

| 工作内容 | 说明 | 价值 |
|---------|------|------|
| **客户信息核实更新** | 核实联系方式、用电地址等信息 | 保证客户档案准确 |
| **用电需求收集** | 了解新装、增容、变更意向 | 业务拓展、提前服务 |
| **诉求意见收集** | 收集客户意见、建议、潜在投诉 | 服务改进、预警 |
| **特殊时期需求** | 了解迎峰度夏、春节等特殊时期用电需求 | 保供电支撑 |

### 二、问题排查类

| 工作内容 | 说明 | 价值 |
|---------|------|------|
| **安全隐患排查** | 私拉乱接、违章用电、消防隐患 | 安全管控 |
| **设备异常发现** | 运行异常、负荷异常、窃电嫌疑 | 降损、防窃电 |
| **通道隐患排查** | 树障、违章建筑、外力破坏 | 可靠性提升 |
| **表计异常检查** | 表计故障、计量异常 | 减少电费差错 |

### 三、巡视检查类

| 工作内容 | 说明 | 价值 |
|---------|------|------|
| **配电设施巡视** | 变压器、开关、电缆等设备状态 | 设备健康管理 |
| **线路通道巡视** | 线路通道状况、树障情况 | 可靠性管理 |
| **表计运行检查** | 表计运行状态、抄表情况 | 计量管理 |
| **台区信息维护** | 台区归属、供电范围核实 | 台区线损管理 |

### 四、服务提供类

| 工作内容 | 说明 | 价值 |
|---------|------|------|
| **政策宣传** | 电价政策、安全用电宣传 | 客户知晓度提升 |
| **用电指导** | 安全用电、节约用电指导 | 服务增值 |
| **电费咨询** | 解答电费疑问、催费提醒 | 减少投诉 |
| **业扩咨询** | 新装、增容等业务咨询 | 业务拓展 |

### 五、故障预处理类

| 工作内容 | 说明 | 价值 |
|---------|------|------|
| **故障预判** | 现场故障初步判断 | 减少误报、加快抢修 |
| **故障报修预处理** | 现场信息收集、客户安抚 | 提升抢修效率 |
| **停电通知送达** | 计划停电通知现场送达 | 客户知情权 |

### 六、协调对接类

| 工作内容 | 说明 | 价值 |
|---------|------|------|
| **物业对接** | 与物业建立日常联系机制 | 协同服务 |
| **社区对接** | 与社区、街道建立联系 | 政企协同 |
| **施工单位对接** | 现场施工协调、安全监督 | 施工安全 |

## ❓ 需要讨论的问题

### 问题1：工作边界如何界定？

- 客户经理 vs 抄表员 vs 用电检查员，职责边界在哪？
- 哪些工作是客户经理"应该做"的？
- 哪些工作是"顺便做"的？
- 哪些工作是"发现后上报"的？

### 问题2：如何平衡工作量？

- 现场时间有限，不可能做所有事情
- 如何确定优先级？
- 如何与日常工作协同？

### 问题3：信息化支撑？

- 这些工作如何记录？
- 信息如何流转？
- 如何形成闭环？

### 问题4：考核评价？

- 哪些工作需要量化考核？
- 如何衡量工作效果？
- 如何激励多做事？

## 🎯 建议：重新定义驻点工作范围

### 方案A：扩展现有三大领域

```
网格服务 → 客户服务类（服务前置 + 信息收集）
用电检查 → 设备管理类（问题排查 + 巡视检查）
应急保供 → 应急响应类（应急预案 + 故障预处理）
```

### 方案B：增加新的领域

```
1. 网格服务（服务前置）
2. 用电检查（问题排查）
3. 应急保供（资源前置）
4. 巡视检查（新增）
5. 信息收集（新增）
6. 协调对接（新增）
```

### 方案C：按工作频次分类

```
日常性工作：
- 网格服务、信息收集、巡视检查

周期性工作：
- 用电检查、隐患排查

应急性工作：
- 应急保供、故障预处理
```
